## 🎯 核心变更

### 1. 目录管理重构

#### 当前设计（有问题）

```
~/.learn-system/          # 全局学习数据（路径控制失败）
├── goals/
│   ├── active/
│   ├── paused/
│   ├── completed/
│   └── archived/
├── progress/
├── bookmarks/
└── cache/
```

**问题**：
- ❌ 路径控制完全失效
- ❌ 模型忽略路径说明，自主决定文件位置
- ❌ 试过相对路径、`~` 符号、硬编码绝对路径 - 全部失败

#### 新设计（推荐）

```
.learning/              # 项目级学习数据（相对于项目根）
├── goals.json        # 所有学习目标（JSON）
├── progress.json     # 所有课程进度（JSON）
├── bookmarks.json   # 所有书签（JSON）
└── cache/            # 课程资料缓存
    └── <course-name>/
```

**优点**：
- ✅ 避免路径冲突（使用相对路径即可）
- ✅ 每个项目独立管理学习数据
- ✅ 技能更简单，不需要硬编码路径
- ✅ 符合项目隔离原则
- ✅ JSON 格式易于解析和管理

**缺点**：
- ⚠️ 不同 Agent 的学习数据不共享
  - 但这是可接受的，因为学习数据通常是项目相关的

### 2. 强制性流程

#### 问题

模型忽略技能中的路径说明，自主决定文件保存位置。

#### 解决方案

##### A. 强制性语言

在 SKILL.md 顶部添加：

```markdown
## 强制性规则 ⚠️

以下规则**必须严格遵守**，违反将被视为技能执行失败：

### 路径规则
- **必须使用**：`.learning/` 作为根目录（相对于项目根）
- **禁止**：使用任何其他路径或自主决定文件位置
- **要求**：任何文件操作前必须先检查目录是否存在

### 创建目录
- **必须**：创建文件前先使用 `exec mkdir -p .learning/<subdir>`
- **禁止**：假设目录已存在或跳过创建

### 错误处理
- 如果路径不符合预期，**必须**立即报错并停止
- 不得尝试"修复"或使用替代路径
```

##### B. 前置操作流程

在 SKILL.md 中添加强制性前置操作：

```markdown
## 所有文件操作的前置步骤 ⚠️

在执行任何文件操作（创建、读取、写入、更新）之前，**必须**按以下顺序执行：

### 步骤 1：确保基础目录存在
```bash
exec: mkdir -p .learning
```

### 步骤 2：确保子目录存在
```bash
exec: mkdir -p .learning/bookmarks
exec: mkdir -p .learning/progress
exec: mkdir -p .learning/cache
exec: mkdir -p .learning/goals
```

### 步骤 3：验证目录结构
```bash
exec: ls -la .learning/
```

**禁止**：跳过以上任何步骤或使用其他路径。
```

##### C. 子模块中的强制性说明

在每个子模块顶部添加：

```markdown
## 强制性路径规则 ⚠️

本模块的所有文件操作**必须使用**：
- 基础路径：`.learning/`
- 书签路径：`.learning/bookmarks/`
- 缓存路径：`.learning/cache/`
- 进度路径：`.learning/progress/`
- 目标路径：`.learning/goals/`

**违反规则将导致技能执行失败。**
```
