# 学习管理系统 (Learn System)

> 本技能提供完整的学习管理功能，包括目标创建、课程学习、进度追踪、书签管理和知识缓存。

## 使用建议

### 最佳实践：使用专门的学习 Agent

**强烈推荐**：创建一个专门用于学习的 Agent，以避免语义冲突。

**为什么？** 本技能使用语义意图匹配，可能会与以下场景冲突：
- 通用的"学习"相关查询（例如"学习这个 bug"）
- 其他使用相似关键词的技能
- 代码审查或调试等场景中出现的"学习"字样

**解决方案**：使用专门的学习 Agent
```bash
openclaw agent create learning-agent --skills learn-system-openclaw
openclaw session learning-agent
```

### 语义匹配的行为说明

主技能使用语义匹配将请求路由到对应的子模块。这种方式提供了灵活性，但偶尔可能会匹配到非学习相关的查询。

为了提高准确性，建议：
- 使用具体的关键词（例如"创建学习目标"而不是"创建目标"）
- 使用专门的学习 Agent（推荐）
- 如果遇到误触发，请反馈以便改进

### 中文用户特别说明

本技能主要面向中文用户设计，所有关键词和提示均为中文。在英文环境使用时，可能需要调整关键词列表。

## 意图识别与路由

当检测到学习相关的语义时，按以下逻辑路由到对应的子模块：

### 1. 学习目标相关

- **创建学习目标**：当用户说"我要学 X"且 X 不在现有课程列表中时
- **查看学习目标**：查询所有目标概览
- **切换/暂停/恢复学习目标**：管理目标状态

→ 路由到 `modules/goal-creator.md` 或 `modules/goal-tracker.md`

### 2. 课程学习相关

- **开始学习**：当用户说"我要学 X"且 X 在现有课程列表中时
- **完成学习**：标记课程为已完成
- **重置课程**：清除课程进度

→ 路由到 `modules/module-manager.md`

### 3. 学习状态相关

- **查看学习状态**：查看所有课程进度概览
- **更新进度**：更新特定课程的进度
- **课程进度**：查看特定课程的详细进度

→ 路由到 `modules/status.md`

### 4. 书签管理相关

- **创建书签**：记录学习中的疑问点
- **继续书签**：回到之前的书签继续探索
- **完成书签**：标记书签为已完成
- **删除书签**：移除不需要的书签

→ 路由到 `modules/tools.md`

### 5. 知识缓存相关

- **刷新缓存**：更新课程知识缓存
- **查看缓存**：查看缓存状态
- **知识缓存**：管理课程资料缓存

→ 路由到 `modules/tools.md`

- "我要学 X" → 检查 X 是否为现有课程

## 注意事项

1. **语义匹配**：本技能使用语义匹配，可能偶尔误触发。使用专门的学习 Agent 可以避免此问题。

2. **目录隔离**：使用 `.learn-system/` 作为根目录，避免与其他技能冲突。

3. **关键词精确性**：子模块使用精确关键词匹配，主技能使用语义匹配。

4. **路由优先级**：
   - "我要学 X" → 先检查课程列表，再决定路由
   - 具体操作（创建/查看/更新）→ 直接路由到对应子模块

5. **子模块协调**：子模块间可能需要协同工作（如目标创建后自动开始学习）。

```python
def detect_intent(user_input):
    # 1. 检测学习相关语义
    if not has_learning_semantic(user_input):
        return None

    # 2. 提取核心动作
    action = extract_action(user_input)  # 创建/查看/更新/删除

    # 3. 提取对象类型
    obj_type = extract_object_type(user_input)  # 目标/课程/进度/书签/缓存

    # 4. 路由到对应子模块
    return route_to_module(action, obj_type)
```
