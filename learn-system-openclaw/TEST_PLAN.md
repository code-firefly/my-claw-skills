# 测试计划 (Test Plan)

## 测试目标

验证 `learn-system-openclaw` 技能的功能完整性和稳定性，确保可以安全发布到 ClawHub。

---

## 测试环境

### 方案 A：Git Worktree（推荐）

**优点**：
- 独立的工作副本，不影响主分支
- 可以并行测试多个版本
- 易于回滚

**步骤**：
```bash
# 1. 确保在主分支
git checkout main

# 2. 创建 worktree
git worktree add /tmp/learn-system-test testing

# 3. 在 worktree 中测试
cd /tmp/learn-system-test

# 4. 测试代码...
# ... 测试逻辑 ...

# 5. 测试完成后
# 如果成功：合并到 main
git checkout main
git merge testing

# 如果失败：删除 worktree
git worktree remove /tmp/learn-system-test
git branch -D testing
```

### 方案 B：沙盒技能目录

**步骤**：
```bash
# 1. 在临时目录创建测试版本
mkdir -p /tmp/test-skills/learn-system-openclaw
cp -r openclaw/learning/learn-system-openclaw/* /tmp/test-skills/learn-system-openclaw/

# 2. 测试时临时添加到 OpenClaw 技能路径
# （需要配置 OpenClaw 的技能搜索路径）
```

### 方案 C：ClawHub Beta 版本测试

**优点**：
- 最接近真实使用场景
- 可以在干净的 OpenClaw 环境测试

**步骤**：
1. 发布到 ClawHub（测试版本号如 `0.1.0-beta.1`）
2. 在另一个环境安装测试
3. 验证功能完整性
4. 确认无误后发布正式版本 `1.0.0`
