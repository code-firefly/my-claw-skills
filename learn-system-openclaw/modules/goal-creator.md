# 学习目标创建 (Goal Creator)

> **精确关键词匹配**：当用户输入包含以下关键词之一时触发此模块

## 🚨 强制性路径规则 ⚠️

**必须**使用 `.learning/` 作为根目录。

**禁止**：
- 使用任何其他路径
- 让模型自主决定文件位置

---

## 触发关键词

- 创建学习目标 / 新建学习目标 / 创建目标 / 新建目标 / 添加目标
- 新学习目标

## 功能

AI 引导用户创建个性化学习目标并生成课程结构。

---

## 前置步骤（所有操作前必须执行）

```bash
# 步骤 1：确保目录存在
exec: mkdir -p .learning

# 步骤 2：读取现有目标（如果文件存在）
read: .learning/goals.json
```

**禁止**：跳过以上任何步骤。

## 目标类型识别

### 类型 1：理论型（📖）

**特征**：用户表达为"学会使用 X"、"了解 X"

**课程结构**：按知识主题划分

### 类型 2：实践型（🛠️）

**特征**：用户表达为"搭建 X"、"构建 X"、"实现 X"

**课程结构**：按项目阶段划分

### 类型 3：混合型（🔄）

**特征**：同时包含学习和实践需求

**课程结构**：理论部分 + 实践部分

## 目标生成算法

### 生成目标 ID

```bash
# 使用主题 + 时间戳生成唯一 ID
GOAL_ID="learn-$(echo $TOPIC | tr '[:upper:]' ' '-').$(date +%s)"
```

### 生成目标结构

```json
{
  "id": "learn-<topic>-<timestamp>",
  "name": "学习 <topic>",
  "status": "active",
  "type": "<type>",  // theory | practice | mixed
  "created_at": "<当前日期 YYYY-MM-DD>",
  "description": "<根据用户输入生成>",
  "priority": "<根据目的确定>",  // P0 必学 | P1 推荐 | P2 可选 | P3 了解
  "course_name": "<课程名称或 null>"
}
```

### 课程结构生成

#### 理论型课程生成

```
输入：用户学习主题 + 知识源分析结果
输出：课程大纲（按主题）

算法：
1. 从核心概念列表中选择主要概念
2. 按"基础 → 进阶 → 高级"排序
3. 每个概念成为一个模块
4. 每个模块包含：
   - 概念定义
   - 使用场景
   - 相关命令/配置
   - 实践示例
```

#### 实践型课程生成

```
输入：项目需求分析
输出：课程大纲（按阶段）

算法：
1. 分析项目开发流程
2. 按阶段划分：规划 → 搭建 → 实现 → 测试 → 部署
3. 每个阶段成为一个模块
4. 每个模块包含：
   - 阶段目标
   - 所需技能
   - 具体任务
   - 验收标准
```

## 步骤 6：确认并保存到 goals.json

### 读取现有数据

```bash
read: .learning/goals.json
```

### 解析 JSON

解析现有 JSON，获取 `goals` 对象。

### 添加新目标

```json
{
  "version": "2.0.0",
  "active_goal_id": "learn-docker-1234567890",
  "goals": {
    "learn-docker": {
      "id": "learn-docker-1234567890",
      "name": "学习 Docker 容器技术",
      "status": "active",
      "type": "practice",
      "created_at": "2026-03-13",
      "description": "掌握 Docker 容器化和编排技能",
      "priority": "P1",
      "course_name": "docker-fundamentals"
    },
    ... // 其他目标
  }
}
```

### 写回文件

```bash
write: .learning/goals.json
```

**重要**：必须完整替换文件内容，不得追加或部分修改。

## ⚠️ 注意事项

### 通用注意事项

1. **路径强制**：必须使用 `.learning/goals.json`，不得使用其他路径
2. **用户确认**：创建目标前必须显示预览并等待用户确认
3. **日期格式**：统一使用 YYYY-MM-DD 格式
4. **错误处理**：如果 JSON 解析失败，报告明确错误并停止

### JSON 操作注意事项

1. **读取 JSON**：使用 `read` 工具读取，然后解析
2. **修改 JSON**：解析后修改 JSON 对象，然后写回
3. **写回 JSON**：使用 `write` 工具完整替换文件
4. **错误处理**：如果 JSON 解析失败，报告错误并停止
