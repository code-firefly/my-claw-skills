# 学习目标追踪 (Goal Tracker)

> **精确关键词匹配**：当用户输入包含以下关键词之一时触发此模块

## 🚨 强制性路径规则 ⚠️

**必须**使用 `.learning/` 作为根目录。

**禁止**：
- 使用任何其他路径
- 让模型自主决定文件位置

---

## 触发关键词

- 查看学习目标 / 显示学习目标 / 学习目标列表
- 我的学习目标 / 所有学习目标
- 学习目标概览 / 学习目标 / 看看学习目标 / 当前学习目标
- 学习目标状态
- 切换学习目标
- 暂停学习目标 / 恢复学习目标
- 归档学习目标
- 完成目标

---

## 功能

管理多个学习目标的生命周期和状态切换。

---

## 前置步骤（所有操作前必须执行）

```bash
# 步骤 1：确保目录存在
exec: mkdir -p .learning

# 步骤 2：读取目标文件
read: .learning/goals.json
```

**禁止**：跳过以上任何步骤。

## 切换学习目标

### 执行步骤

**步骤 1**：读取 `.learning/goals.json`

**步骤 2**：显示所有学习目标列表

**步骤 3**：用户选择目标

**步骤 4**：更新目标状态和 `active_goal_id`

```json
{
  "active_goal_id": "learn-docker",
  "goals": {
    "learn-docker": {
      "status": "active",
      ...
    },
    "learn-openclaw": {
      "status": "paused",
      ...
    }
  }
}
```

**步骤 5**：写回 `.learning/goals.json`

**步骤 6**：显示切换结果

```
✅ 已切换学习目标

激活目标：学习 Docker 容器技术
暂停目标：学会使用 OpenClaw
```

## 暂停学习目标

**步骤 1**：读取 `.learning/goals.json`

## ⚠️ 注意事项

### 通用注意事项

1. **路径强制**：必须使用 `.learning/goals.json`，不得使用其他路径
2. **用户确认**：切换/暂停/恢复/归档/完成操作前，总是先显示预览并等待用户确认
3. **日期格式**：统一使用 YYYY-MM-DD 格式
4. **错误处理**：如果 JSON 解析失败，报告明确错误并停止

### JSON 操作注意事项

1. **读取 JSON**：使用 `read` 工具读取，然后解析 JSON
2. **修改 JSON**：解析后修改 JSON 对象，然后写回
3. **写回 JSON**：使用 `write` 工具完整替换文件
4. **错误处理**：如果 JSON 解析失败，报告明确错误并停止
