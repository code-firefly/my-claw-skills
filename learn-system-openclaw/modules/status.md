## 更新进度

**触发方式**：
- "更新进度 `<课程名>`" / "更新 `<课程名>` 的学习进度" / "同步进度 `<课程名>`"
- "记录进度 `<课程名>`" / "保存进度 `<课程名>`" / "打勾 `<课程名>`"
- "标记完成项 `<课程名>`" / "更新 `<课程名>` 状态" / "刷新进度 `<课程名>`"
- "`<课程名>` 进度更新" / "帮我更新 `<课程名>` 进度" / "`<课程名>` 学了多少"

**执行步骤**：

### 步骤 1：读取课程的 `checklist.md`

### 步骤 2：解析学习模式和完成项：
   - 检测已勾选的学习模式（快速/完整）
   - 统计已完成的 checkbox 数量
   - 根据学习模式计算对应的完成百分比

### 步骤 3：读取 `PROGRESS.md` 获取当前进度

### 步骤 4：生成更新预览：
   ```
   Progress: X% -> Y%
   Status: 进行中
   进度条: ██████░░░░░░░░░░░░░░ 30%
   ```

### 步骤 5：用户确认后更新 `PROGRESS.md`：
   - 更新课程的完成百分比
   - 添加学习日志条目

**进度计算规则**：
- 快速模式：只统计"📚 快速模式"章节下的 checkbox
- 完整模式：统计"🛠️ 完整模式"章节下的所有 checkbox
- 未选择模式：基于完整模式计算

```markdown
# 更新进度

# 添加学习日志
| 2026-03-07 | ai-tools-fundamentals | 更新进度 | 完成第一部分 |
```

## 输出格式规范

### 进度条
```
███░░░░░░░░ 30%
██████░░░░░ 60%
██████████░ 80%
████████████ 100%
```

### 状态徽章
- 🟢 未开始
- 🟡 进行中
- ✅ 已完成

### 优先级标识
- P0 🔴 必学
- P1 🟠 推荐
- P2 🟡 可选
- P3 ⚪ 了解

---

## 注意事项

1. **进度计算**：基于实际勾选的 checkbox 数量，不要估算
2. **模式识别**：准确识别用户选择的学习模式
3. **用户确认**：更新进度前显示预览
4. **别名处理**：支持旧名称和别名
