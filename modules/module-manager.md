# 课程管理 (Module Manager)

> **精确关键词匹配**：当用户输入包含以下关键词之一时触发此模块

## 触发关键词

- 开始学习课程 / 开始学习 X
- 学习 X / 学 X
- 完成学习 / 完成课程
- 重置课程

## 功能

管理课程学习的生命周期。

---

## 开始学习

**执行步骤**：

### 步骤 1：检查知识缓存

1. 读取模块的 `knowledge/.metadata.json`（如果存在）

2. **情况 A：缓存存在**
   - 显示：`📦 发现本地缓存 (缓存日期: YYYY-MM-DD)`
   - 询问用户：
     - 1. 使用本地缓存（快速开始）
     - 2. 更新缓存（拉取最新资料）
     - 3. 跳过缓存（直接开始学习）

3. **情况 B：缓存不存在**
   - 显示：`⚠️  未找到知识缓存`
   - 询问用户：
     - 1. 初始化缓存（推荐，首次拉取）
     - 2. 跳过缓存（直接开始学习）

### 步骤 2：模块名称别名解析

如果用户输入的模块名不在映射表中，检查别名映射：

| 旧名称 | 新名称 | 说明 |
|--------|--------|------|
| `claude-code-core` | `ai-tools-fundamentals` | AI 工具基础 |
| `mcp-basics` | `mcp-protocol` | MCP 协议入门 |
| `agent-sdk` | `agent-configuration` | Agent 配置与使用 |
| `mcp-advanced` | `mcp-advanced-config` | MCP 高级配置 |
| `openclaw-ecosystem` | `ai-orchestration` | AI 编排平台 |
| `everything-claude-code` | `ai-resources-research` | AI 资源研究 |
| `cc-switch` | `config-management` | 配置管理工具 |
| `spec-kit` | `spec-driven-dev` | 规范驱动开发 |
| `projects` | `practical-projects` | 综合实战项目 |

如果找到别名：
- 显示提示：`💡 注意：'{旧名称}' 已更名为 '{新名称}'，正在使用新名称继续...`
- 使用新名称继续后续流程

### 步骤 3：查找课程路径

从课程映射表查找课程路径。

### 步骤 4：验证课程状态

检查课程是否已开始学习（检查 checklist.md 的学习模式选择）
- 如果已开始，显示警告信息：
  ```
  ⚠️  该课程已经开始了！
  当前状态：{状态} {进度}
  是否要重新开始？(y/n)
  ```

### 步骤 5：选择学习模式

询问用户选择：
- 1. 📚 快速模式（约 3 天）- 理论为主
- 2. 🛠️ 完整模式（约 1 周）- 理论+实践

**模式说明**：
- 快速模式：适合快速了解概念，以阅读文档为主
- 完整模式：深入掌握技能，包含实践练习

### 步骤 6：显示更新预览

```

║                  开始学习预览                          ║
╠═══════════════════════════════════════════════════════╣
║ 课程：ai-tools-fundamentals                           ║
║ 优先级：P0 🔴 必学                                     ║
║ 模式：📚 快速模式（约 3 天）                           ║
║ 开始日期：2026-03-07                                   ║
╠═══════════════════════════════════════════════════════╣
║ 将要更新：                                             ║
║   ✓ 01-基础入门/ai-tools-fundamentals/checklist.md    ║
║   ✓ 01-基础入门/ai-tools-fundamentals/notes.md        ║
║   ✓ PROGRESS.md                                        ║
╠═══════════════════════════════════════════════════════╣
║ 确认开始学习？(y/n)                                    ║
╚═══════════════════════════════════════════════════════╝
```

### 步骤 7：更新文件

用户确认后，更新以下文件：

**checklist.md**：
```markdown
# 勾选对应的学习模式
- [x] **快速模式**（约 3 天）- 理论为主
- [ ] **完整模式**（约 1 周）- 理论+实践
```

**notes.md**：
```markdown
> **学习模式**：快速模式
> **开始日期**：2026-03-07
```

**PROGRESS.md**：
```markdown
# 更新课程行
| ai-tools-fundamentals | P0 🔴 | 快速模式 | 进行中 | 0% |

# 添加学习日志
| 2026-03-07 | ai-tools-fundamentals | 开始学习 | 选择快速模式 |
```

### 步骤 8：显示成功信息

```
✅ 学习已开始！

📅 开始日期：2026-03-07

下一步：
• 查看 README.md 了解课程概述
• 按照 checklist.md 逐项完成学习
• 随时说"更新进度"同步进度

💡 提示：说"查看 ai-tools-fundamentals 状态"可查看详细进度
```

## 注意事项

1. **用户确认**：修改多个文件前，总是先显示预览并等待用户确认
2. **日期格式**：统一使用 YYYY-MM-DD 格式
3. **文件路径**：所有路径相对于仓库根目录
4. **错误处理**：如果文件不存在或课程未找到，给出明确的错误提示
5. **进度计算**：基于实际勾选的 checkbox 数量，不要估算
6. **别名处理**：用户使用旧名称时，友好提示并使用新名称继续
