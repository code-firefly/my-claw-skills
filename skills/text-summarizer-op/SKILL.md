# Text Summarizer (OpenClaw Optimized)

智能文本摘要生成技能，能够快速将长文本压缩为简洁的摘要，提取关键信息。

## 使用场景

当用户提到以下需求时触发：
- "总结这篇文章"
- "帮我摘要一下"
- "提取关键点"
- "生成概要"
- "缩短这段文本"
- "简要说明"
- "压缩内容"

## 摘要生成策略

### 摘要长度控制

根据用户需求或文本长度自动调整摘要：

**短摘要（1-2句话）：**
- 适用于快速浏览
- 只保留核心信息
- 省略细节

**中等摘要（3-5句话）：**
- 适用于常规需求
- 包含主要观点
- 适当保留支撑信息

**详细摘要（段落形式）：**
- 适用于需要深度理解的场景
- 保留重要细节
- 结构化展示信息

### 摘要类型

**列表式摘要：**
```
要点1
要点2
要点3
```

**段落式摘要：**
连续的段落，逻辑连贯

**结构化摘要：**
```
核心观点：...
关键信息：...
结论：...
```

## 工作流程

1. **理解用户意图**
   - 读取用户提供的文本
   - 确定摘要长度和类型需求
   - 识别文本类型（技术文档、博客、邮件等）

2. **分析文本结构**
   - 识别标题、段落、列表
   - 提取关键句和关键词
   - 理解段落之间的逻辑关系

3. **生成摘要**
   - 保留核心信息和重要细节
   - 保持原文的逻辑结构
   - 使用简洁清晰的语言
   - 确保摘要的准确性和完整性

4. **验证质量**
   - 检查是否遗漏关键信息
   - 确保没有添加无关内容
   - 验证摘要长度是否合理

## 最佳实践

- **保持客观**：不添加原文没有的信息
- **保留关键信息**：重要数据、结论、决策不应遗漏
- **使用原文术语**：技术术语、专有名词保持一致
- **结构清晰**：使用适当的分段和列表
- **简洁明了**：避免冗余表达

## 输出格式

根据原文类型自动选择合适的格式：

**技术文档：** 结构化摘要 + 关键要点列表
**博客文章：** 段落式摘要 + 主要观点
**会议记录：** 议题 + 讨论要点 + 结论
**邮件内容：** 核心信息 + 行动项
